## 3. 音声化（TTS）

### Q6: 音声合成エンジンの希望は？


| エンジン         | 特徴                           |
| ------------------ | -------------------------------- |
| OpenAI TTS       | 高品質、日本語対応、API 課金   |
| Google Cloud TTS | 高品質、多言語、API 課金       |
| Azure TTS        | 高品質、多言語、API 課金       |
| VOICEVOX         | 無料、ローカル実行、日本語特化 |
| 特にこだわりなし | コスト・品質のバランスで選定   |

→ その他: ひとまず `edge-tts` をデフォルトとして実装してください。ただし、将来的なOSS化や商用化を見据え、**TTS機能は「ストラテジーパターン（プラグインアーキテクチャ）」で設計** してください。後から `Kokoro TTS` や `OpenAI TTS` などの別クラスを簡単に追加でき、`config.yaml` の設定ひとつで切り替えられるようにインターフェース（基底クラス）を定義しておくことが絶対条件です。
